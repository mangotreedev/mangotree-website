    A SMALL INTRO TO TAILWINDCSS

    Written by Nico Proto

    Feb 20 ¬∑ 7 min read

    ### Intro In this small intro, I will help you understand how to think in TailwindCSS. We‚Äôll go through a small theory intro ü§ì(sorry in advance) and will jump next to a more hands-on üëã exercise to practice what we learned. ### Why use Tailwind * **Simple and specific:** Most of the utilities included have specific functionality and don‚Äôt add extra styling to them (contrary to Bootstrap). * **Highly customizable:** Although it has a default configuration, we can simply override it in the `tailwind.config.js` file. * **Can be optimized:** We can use PurgeCSS in production to remove any classes we don‚Äôt use from Tailwind, only loading utilities we use. * **Easy to write:** As soon as you get used to it, it becomes super fast to write your CSS classes; you can also group classes into components by using `@apply`. * [other reasons](https://dev.to/joserfelix/why-use-tailwind-css-for-your-next-project-39hp) ### Some comments before we start * TailwindCSS doesn‚Äôt have JavaScript included, it is pure CSS. So if we want to make things dynamic we need to write our own JavaScript. * I will assume you know the basics of CSS üòÑ. * We won‚Äôt go through the installation of TailwindCSS, for that feel free to check their [installation guide](https://tailwindcss.com/docs/installation). * File locations are considered for Ruby on Rails. ### Configuration file TailwindCSS configuration lives in a file called `tailwind.config.js`. In this file, we can customize every TailwindCSS variable. ### How TailwindCSS works * **Utility Classes:** As with most frameworks, TailwindCSS uses utility classes to style HTML elements; its naming convention is super intuitive (`text-center`, `font-bold`, etc.). * **Prefixes:** We can add prefixes on different classes to make them work conditionally on different breakpoints or states. _For example:_ `_hover:font-bold_` _will change the font of the element to bold when hovered._ ### Prefixes **Responsive Design [(see docs)](https://tailwindcss.com/docs/responsive-design)** On TailwindCSS classes we can add a **breakpoint prefix** which will make it work conditionally at different breakpoints. **Usage** ```html <div class="md:w-92 lg:w-40">   [...] <div class="sm:text-center lg:text-left"> ``` ### Hover, Focus & other states As with the **responsive design**, we can use prefixes to style different states of our elements. **Usage** ```html <button class="bg-purple-600 hover:bg-purple-800">   Sign up </button> ``` ### Utility Classes Writing **TailwindCSS** utility classes keeps the same structure among most CSS properties. We‚Äôll go through some examples to clarify how the structure works, but feel free to find any specific utility class on their amazing [**documentation**](https://tailwindcss.com/docs). ### Fonts [**Font Family**](https://tailwindcss.com/docs/font-family) **‚Üí** `font-family` **‚Üí** `font-{type}` ```html <p class="font-sans ..."> <p class="font-serif ..."> <p class="font-mono ..."> ``` _Note:_ `font-sans`_,_ `font-serif` _and_ `font-mono` _are variables that we can change in our config file to the_ `font-family` _of our choice._ [**Font Size**](https://tailwindcss.com/docs/font-size) **‚Üí** `font-size` **‚Üí** `text-{size}` ```html <p class="text-xs ...">The quick brown fox ...</p> <p class="text-xl ...">The quick brown fox ...</p> ``` [**Font Weight**](https://tailwindcss.com/docs/font-weight) **‚Üí** `font-weight` **‚Üí** `font-{weight}` ```html <p class="font-bold ...">The quick brown fox ...</p> <p class="font-thin ...">The quick brown fox ...</p> ``` [**Text Align**](https://tailwindcss.com/docs/text-align) **‚Üí** `text-align` **‚Üí** `text-{alignment}` ```html <p class="text-left ...">Lorem ipsum dolor sit amet ...</p> <p class="text-center ...">Lorem ipsum dolor sit amet ...</p> ``` ### Container The `container` class will fix the element width depending on the current breakpoint. [Container](https://tailwindcss.com/docs/container) **‚Üí** `width` ```html <div class="container">   <!-- ... --> </div> ``` ### Margin & Padding [**Margin**](https://tailwindcss.com/docs/margin) **‚Üí** `margin` **‚Üí** `m{direction}-{size}` ```html <div class="mt-8 ...">mt-8</div> <div class="m-5 ...">m-5</div> ``` Left + Right ‚Üí `mx` Top + Down ‚Üí `my` [**Padding**](https://tailwindcss.com/docs/padding) **‚Üí** `padding` **‚Üí** `p{direction}-{size}` ```html <div class="pb-3 ...">pb-4</div> <div class="py-2 ...">py-2</div> ``` Left + Right ‚áí `px` Top + Down ‚áí `py` ### Colors The color class utilities allow us to use the same kind of syntax to style different properties. `{property}-{color}-{value}` ```html <p class="text-purple-600 ..."></p> <button class="bg-green-500 ...">Button</button> <input class="border-2 border-red-500 ..."> ``` ### Width and Height [**Height**](https://tailwindcss.com/docs/height) **‚Üí** `height` **‚Üí** `h{direction}-{size}` ```html <div class="h-12 ..."></div> <div class="h-16 ..."></div> ``` [**Width**](https://tailwindcss.com/docs/width) **‚Üí** `width` **‚Üí** `w{direction}-{size}` ```html <div class="w-20 ..."></div> <div class="w-28 ..."></div> ``` ### Display No rocket science here, just classes that apply different display properties. `flex` **‚Üí** `display: flex` `inline-block` **‚Üí** `display: inline-block` `hidden` **‚Üí** `display: none` ### Creating components [(see more)](https://tailwindcss.com/docs/extracting-components) Sometimes when repeating classes over and over on the same elements (buttons, avatars, etc.), it‚Äôs better to refactor our code creating components that help us to update and reuse our code. **TailwindCSS** includes the `@apply` directive that extracts common utility patterns to CSS component classes. ```scss .btn-indigo {   @apply py-2 px-4 bg-indigo-500 text-white font-semibold rounded-lg shadow-md hover:bg-indigo-700 focus:outline-none focus:ring-2 focus:ring-indigo-400 focus:ring-opacity-75; } ``` ```html <button class="btn-indigo">   Click me </button> ``` TailwindCSS also recommends wrapping the styles where we use `@apply` with `@layer components { }` so it can recognize that those styles belong to the `components`([performance and organization reasons](https://tailwindcss.com/docs/extracting-components)). ### Enough theory! If the image is a square it will create a circle. * `h-24` **&** `w-24` **‚Üí** `weight` **and** `height` * `inline-block` **‚Üí** `display` ### Buttons Ok, now let‚Äôs do a primary and secondary `background-color` on `:hover` * `focus:outline-none` **‚Üí** `background-color` on `:focus` * `text-white` **‚Üí** `color` * `rounded` **‚Üí** `border-radius` * `border` **‚Üí** `border-width` * `transition` **‚Üí** `transition` (with a transition delay of 150ms) * `duration-300` **‚Üí** `transition-duration` * `inline-block` **‚Üí** `display` ### Banner
